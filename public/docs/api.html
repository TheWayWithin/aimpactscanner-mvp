Authentication

All API requests require authentication via an API key. Generate keys from your account dashboard.

Include your key in the X-API-Key header:

curl -H "X-API-Key: ais_sk_your_key_here" \
  https://aimpactscanner-backend-production.up.railway.app/api/analyze
API keys are shown only once when created. Store them securely. If lost, revoke and create a new one.

Endpoints

Base URL: https://aimpactscanner-backend-production.up.railway.app

Analysis

POST /api/analyze

Run a synchronous AI-readiness analysis on a URL.

Request Body

FieldTypeDescription
urlstringURL to analyze (required)
userIdstringYour user ID (required)
userTierstringYour tier, e.g. "scale"

Response

{
  "success": true,
  "overall_score": 73,
  "factors": [
    { "name": "Structured Data", "score": 85, "weight": 0.15 },
    { "name": "Content Quality", "score": 70, "weight": 0.20 }
  ],
  "analysisId": "uuid"
}
POST /api/analyze/async

Start an async analysis. Returns a job ID for polling.

FieldTypeDescription
urlstringURL to analyze (required)
userIdstringYour user ID (required)
analysisIdstringPre-created analysis record ID
userTierstringYour tier
{ "success": true, "jobId": "uuid", "analysisId": "uuid" }
GET /api/analyze/jobs/:jobId

Check the status of an async analysis job.

// Pending
{ "status": "processing" }

// Complete
{ "status": "completed", "result": { "overall_score": 73, ... } }

// Failed
{ "status": "failed", "error": "reason" }

API Key Management

Manage your API keys. These endpoints require JWT authentication (browser session), not API key auth.

POST /api/keys

Create a new API key. The raw key is returned only once.

FieldTypeDescription
namestringHuman-readable name (required, max 64 chars)
{
  "id": "uuid",
  "name": "My Script",
  "key": "ais_sk_abc123...",
  "prefix": "ais_sk_abc12",
  "created_at": "2026-02-10T...",
  "warning": "Save this key now. It cannot be shown again."
}
GET /api/keys

List all your API keys (active and revoked).

{
  "keys": [
    {
      "id": "uuid",
      "name": "CI Pipeline",
      "key_prefix": "ais_sk_abc12",
      "created_at": "2026-02-10T...",
      "last_used_at": "2026-02-10T...",
      "is_active": true,
      "revoked_at": null
    }
  ]
}
DELETE /api/keys/:id

Revoke an API key. Soft-deletes the key (cannot be undone).

{ "success": true, "message": "API key revoked" }

Code Examples

curl

# Run analysis
curl -X POST \
  https://aimpactscanner-backend-production.up.railway.app/api/analyze \
  -H "Content-Type: application/json" \
  -H "X-API-Key: ais_sk_your_key_here" \
  -d '{"url": "https://example.com", "userId": "your-user-id", "userTier": "scale"}'

Python

import requests

API_KEY = "ais_sk_your_key_here"
BASE = "https://aimpactscanner-backend-production.up.railway.app"

# Run analysis
resp = requests.post(f"{BASE}/api/analyze", json={
    "url": "https://example.com",
    "userId": "your-user-id",
    "userTier": "scale"
}, headers={"X-API-Key": API_KEY})

result = resp.json()
print(f"Score: {result['overall_score']}")

JavaScript (Node.js)

const API_KEY = "ais_sk_your_key_here";
const BASE = "https://aimpactscanner-backend-production.up.railway.app";

const resp = await fetch(`${BASE}/api/analyze`, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "X-API-Key": API_KEY,
  },
  body: JSON.stringify({
    url: "https://example.com",
    userId: "your-user-id",
    userTier: "scale",
  }),
});

const result = await resp.json();
console.log(`Score: ${result.overall_score}`);
