🔍 Tab Visibility Fix Test

Test Scenario

This test simulates the dashboard white-out issue when clicking external links.

  1. Open the AImpactScanner dashboard in another tab
  2. Sign in and navigate to the dashboard
  3. Click one of the test links below to open an external site
  4. Return to the dashboard tab
  5. Check if the dashboard displays correctly without white-out

Test Links (Open in New Tab)

Open FreecalcHub.com Open Example.com Open Google.com

Expected Console Output

When switching tabs, you should see these console logs:

🔍 Tab visibility changed: HIDDEN (change #1)
💨 Window lost focus
⏱️ Tab was hidden for XXXms
🔍 Tab visibility changed: VISIBLE (change #2)
🎯 Window gained focus
👁️ Tab not visible - skipping auth state change processing
👁️ Tab not visible - skipping history load
👁️ Tab recently hidden - skipping user tier fetch to prevent duplicates
🔄 useUserInitializer: Skipping duplicate initialization for user: XXX

✅ Success Indicators

  • Dashboard loads without white-out when returning from external tab
  • No duplicate "Loading analysis history" messages in console
  • UserInitializer doesn't re-trigger when returning to tab
  • No duplicate API calls visible in Network tab
  • Smooth transition when switching between tabs

⚠️ Previous Issues (Should Not Occur)

  • White-out/blank screen when returning to dashboard
  • Multiple "Loading analysis history" messages
  • "Skipping duplicate initialization" appearing multiple times
  • 404 errors for get_user_data RPC function
  • Duplicate network requests when tab regains focus
