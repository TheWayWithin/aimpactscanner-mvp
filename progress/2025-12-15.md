## Technical Decisions

### Why Sync Over Async (For Now)

**Problem**: Async job queue requires background worker process
**Decision**: Use sync endpoint for MVP, async ready for scale
**Rationale**:
- Sync endpoint processes immediately (no queue delay)
- Works for current traffic levels
- Async infrastructure in place when needed for CSR (Puppeteer) analysis

### Why Railway Over Edge Functions

**Problem**: Supabase Edge Functions have 60-second timeout limit
**Decision**: Migrate to Railway Node.js backend
**Benefits**:
- No timeout limits for long-running Puppeteer renders
- Full Node.js ecosystem (vs Deno limitations)
- Better debugging and logging
- Future-proofed for CSR website analysis
