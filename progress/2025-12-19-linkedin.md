We just shipped the biggest update to AImpactScanner since launch.

The problem we solved:

Modern React, Vue, and Angular sites were invisible to our scanner. JavaScript renders the content, but we only saw the empty HTML shell.

The solution:

Full JavaScript rendering. We rebuilt our entire backend to analyze websites the way browsers actually see them.

What this means for you:

- 27 assessment factors (up from 18)
- 9 pillars including new Performance metrics
- JavaScript rendering for SPAs
- LLMs.txt file generation built-in

That last one matters.

LLMs.txt is becoming the robots.txt for AI. It tells ChatGPT, Claude, and Perplexity how to interact with your site.

As AI becomes the primary way people discover businesses, having an optimized llms.txt becomes as important as SEO itself.

Now you can generate one directly from your AImpactScanner analysis.

The technical story:

We migrated from Supabase Edge Functions to Railway Node.js with Puppeteer for headless browser rendering. This removed the 60-second timeout limitation and enabled proper SPA analysis.

Building in public means showing the messy parts. This migration took weeks of debugging JWT token issues, regex timeouts, and environment separation.

Worth it? Absolutely.
