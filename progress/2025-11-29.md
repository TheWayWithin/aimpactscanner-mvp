### 🐛 Issues & Learnings

#### Issue: userTier always showing 'free' in LLMsTxtPanel
- **Root Cause**: App.jsx was not passing `user` prop to SimpleResultsDashboard, so `user?.tier` always returned undefined, defaulting to 'free'
- **Fix**: Added `user={{ tier: userTier }}` prop to SimpleResultsDashboard in App.jsx
- **Prevention**: When adding new components that need user context, verify the prop chain from App.jsx
- **Commit**: `fa298f8`

#### Issue: Edge Function returning 400 Bad Request
- **Root Cause**: Frontend was passing action as query parameter (`generate-llmstxt?action=analyze`) but Supabase SDK's `functions.invoke()` doesn't support query params in function name
- **Fix**: Updated frontend to pass action in request body, updated Edge Function to read `requestBody.action` instead of URL params
- **Prevention**: Always pass data to Edge Functions via request body, not query parameters
- **Commit**: `e35ba76`

#### Issue: Edge Function returning 401 Unauthorized
- **Root Cause**: Auth pattern was creating a second Supabase client with headers, which doesn't properly verify JWT tokens
- **Fix**: Changed to `supabaseAdmin.auth.getUser(jwt)` pattern - extract Bearer token and pass directly to getUser()
- **Prevention**: Use `getUser(jwt)` pattern for Edge Function authentication, not client configuration with headers
- **Commit**: `aaa7cfe`

#### Issue: Staging deployments not appearing
- **Root Cause**: Pushed to `main` branch but staging deploys from `develop` branch
- **Fix**: Merged main into develop and pushed
- **Prevention**: Remember to merge to `develop` for staging deploys
