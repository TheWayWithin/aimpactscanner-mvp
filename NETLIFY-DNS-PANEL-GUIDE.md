## Part 2: Standard DNS Record Values for Resend

**IMPORTANT**: These are the typical values, but you MUST get exact values from your Resend dashboard after adding your domain.

### Standard SPF Record
- **Type**: TXT
- **Name**: `@` (represents root domain)
- **Value**: `v=spf1 include:_spf.resend.com ~all`
- **TTL**: 3600 (default)

### DKIM Record (Example Format)
- **Type**: TXT  
- **Name**: `[unique-selector]._domainkey` (Resend provides exact name)
- **Value**: `v=DKIM1; k=rsa; p=[long-key-string]` (Resend provides exact value)
- **TTL**: 3600 (default)

### DMARC Enhancement (Optional)
- **Type**: TXT
- **Name**: `_dmarc`
- **Value**: `v=DMARC1; p=none; rua=mailto:dmarc@resend.com`
- **TTL**: 3600 (default)

**CRITICAL**: The exact DKIM name and value are unique to your domain and generated by Resend. You MUST copy them from your Resend dashboard.

## Part 3: Adding Records in Netlify DNS Panel

### Adding SPF Record

1. **Click "Add new record"** button in DNS panel
2. **Record type**: Select **"TXT"** from dropdown
3. **Name**: Enter `@` (this represents your root domain)
4. **Value**: Paste: `v=spf1 include:_spf.resend.com ~all`
5. **TTL**: Leave default (3600 seconds)
6. **Click "Save"**

**Visual Guide**:
```
┌─────────────────────────────────────┐
│ Add new record                      │
├─────────────────────────────────────┤
│ Type: [TXT          ▼]              │
│ Name: [@                          ] │
│ Value: [v=spf1 include:_spf.resend.com ~all] │
│ TTL:  [3600                       ] │
│                                     │
│           [Save]    [Cancel]        │
└─────────────────────────────────────┘
```

### Adding DKIM Record

1. **Click "Add new record"** again
2. **Record type**: Select **"TXT"**
3. **Name**: Enter the DKIM selector from Resend (example: `re12345._domainkey`)
4. **Value**: Paste the complete DKIM key from Resend
5. **TTL**: Leave default (3600)
6. **Click "Save"**

**Example DKIM Entry**:
```
Name: re12345._domainkey
Value: v=DKIM1; k=rsa; p=MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8A...
```

### Adding DMARC Enhancement (Optional)

1. **Click "Add new record"** once more
2. **Record type**: Select **"TXT"**
3. **Name**: Enter `_dmarc`
4. **Value**: Enter `v=DMARC1; p=none; rua=mailto:dmarc@resend.com`
5. **TTL**: Leave default (3600)
6. **Click "Save"**

### Common Netlify DNS Panel Issues

**"Go to DNS panel" button missing**:
- Ensure domain is properly connected to Netlify
- Check that nameservers are set to Netlify's nameservers
- Try refreshing the Domain Management page

**"Invalid record" error**:
- Check that TXT record values don't have extra quotes
- Ensure Name field is exactly as specified
- Verify Value field contains complete record

**"TTL too low" warning**:
- Change TTL to 3600 or higher
- Lower TTLs can cause resolution issues

**Record not saving**:
- Check for special characters in Name or Value
- Ensure Value doesn't exceed character limits
- Try logging out and back into Netlify

### DNS Propagation Checking

After adding records, check propagation:

**Online Checker**:
1. Go to https://whatsmydns.net
2. Enter: `aimpactscanner.com`
3. Select: **TXT** record type
4. Look for your new records appearing globally

**Command Line Check**:
```bash
# Check SPF record
dig TXT aimpactscanner.com

# Check DKIM record (replace with actual selector)
dig TXT re12345._domainkey.aimpactscanner.com

# Check DMARC record
dig TXT _dmarc.aimpactscanner.com
```
